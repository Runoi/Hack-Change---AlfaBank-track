# Frontend: Income Prediction Service (Alfa-Bank)

Этот проект представляет собой пользовательский интерфейс (UI) для ML-сервиса прогнозирования дохода.
Стек: **React + TypeScript + Vite**.

## Подготовка к работе

### Требования
- **Node.js**: `v18.x` или выше.
- **npm**: `v9.x` или выше.

### Установка зависимостей
Перейдите в директорию `frontend` и выполните команду:
```bash
npm install
```
Эта команда скачает все необходимые библиотеки, указанные в `package.json`.

---

## Запуск приложения

Существует два основных сценария запуска:

### 1. Локальная разработка (без Docker)

Этот режим подходит для быстрой разработки и отладки UI. Предполагается, что **бэкенд-сервис уже запущен отдельно** на вашем компьютере.

**Шаг 1: Настройка API**
- Откройте файл `.env`, который лежит в корне папки `frontend`.
- Убедитесь, что переменная `VITE_API_URL` указывает на адрес вашего запущенного Python-сервиса (FastAPI).

```
# Пример содержимого .env
VITE_API_URL=http://localhost:8000
```
*Если ваш бэкенд запущен на другом порту, измените `8000` на соответствующий.*

**Шаг 2: Запуск dev-сервера**
Выполните команду:
```bash
npm run dev
```
После успешного запуска в консоли появится ссылка, обычно `http://localhost:5173`. Откройте её в браузере.

### 2. Запуск в составе Docker-экосистемы

Этот режим предназначен для **полноценного тестирования и демонстрации** всего приложения (Frontend + Backend). Он автоматически собирает и связывает оба сервиса.

**Предварительные требования:**
- **Docker** и **Docker Compose** должны быть установлены.

**Шаг 1: Настройка API**
- Откройте файл `.env` в папке `frontend`.
- Измените `VITE_API_URL` так, чтобы он указывал на относительный путь `/api`.

```
# Пример содержимого .env для Docker
VITE_API_URL=/api
```
*Это необходимо, чтобы запросы из браузера шли на Nginx-прокси внутри Docker, а не напрямую на `localhost`.*

**Шаг 2: Сборка и запуск**
- Перейдите в **корневую директорию всего проекта** (где лежит `docker-compose.yml`).
- Выполните команду:
```bash
docker-compose up --build
```
Эта команда соберет Docker-образы для бэкенда и фронтенда, запустит их и свяжет между собой.
После запуска откройте в браузере `http://localhost`.

### Сборка для продакшена (Build)

Чтобы скомпилировать приложение в статические HTML/CSS/JS файлы для деплоя, выполните:
```bash
npm run build
```
Результат будет помещен в папку `dist`
